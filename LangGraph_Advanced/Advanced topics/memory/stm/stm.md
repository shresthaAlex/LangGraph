
## Implementing Short Term Memory in LangGraph

Short Term Memory (STM) using LangGraph concept revolves around maintaining a conversation buffer to handle context effectively, especially when interacting with a Large Language Model (LLM). Each LLM has a context window, which defines the maximum number of tokens it can process at a time. If the input tokens exceed this limit, the LLM's response can become inaccurate, leading to issues such as hallucinations or improper answers .

### Context Overflow Problem

The context overflow problem arises when the number of input tokens sent to the LLM exceeds its context window. This can happen when the entire conversation history is sent with each new user message, especially in long conversations with hundreds of messages . To mitigate this, techniques like trimming and summarization are employed.

### Trimming Technique

Trimming involves setting a maximum token limit for messages sent to the LLM. For instance, if a limit of 500 tokens is set, any messages exceeding this limit will be trimmed to ensure that only the most recent messages are sent . This approach assumes that the most recent messages are more relevant for the ongoing conversation, while older messages can be ignored .

### Summarization Technique

Summarization is a more sophisticated technique that retains the context of previous messages while also sending the latest messages to the LLM. Instead of ignoring older messages, a summary of them is generated and sent along with the recent messages. This way, the LLM maintains a comprehensive understanding of the conversation .

When implementing summarization, it is crucial to delete the older messages from the state after generating their summary to avoid confusion . This ensures that only the summary and the latest messages are retained in the state, allowing for efficient memory management .

### Implementation Steps

1.  **Setting Up Context Management**: The first step involves creating a context manager that handles the state of the conversation. This includes defining a state variable that stores the messages .
    
2.  **Using Checkpoints**: Checkpoints are used to store the state of the conversation at various points, ensuring that the context can be retrieved even after the program restarts .
    
3.  **Database Persistence**: Instead of relying solely on RAM, which is volatile, the state can be stored in a production-grade database like PostgreSQL. This ensures that the conversation history is preserved across sessions .
    
4.  **Code Implementation**: The code for implementing STM in LangGraph involves importing necessary libraries, defining the state, and creating nodes for chat and summarization. The logic for trimming and summarization is integrated into the workflow
    
    
    
5.  **Testing the Implementation**: After setting up the context management and memory techniques, the implementation is tested to ensure that it functions correctly and efficiently handles memory without exceeding the context window 
    

By following these steps, developers can effectively implement Short Term Memory in LangGraph, ensuring that LLMs can maintain context over extended conversations without losing relevant information. This approach enhances the reliability and accuracy of responses generated by the LLM .
