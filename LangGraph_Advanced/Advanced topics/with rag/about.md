## RAG Implementation Reasons
- RAG addresses three main issues: outdated knowledge due to LLMs having a knowledge cut-off date, privacy concerns where users may want to query private data not available during training, and hallucination where LLMs produce false information confidently. 
- By utilizing RAG, LLMs can be connected to private documents, ensuring accurate answers based on user-specific data and mitigating the risk of hallucination by grounding responses in provided information.
-  These enhancements are essential for effectively using LLMs in sensitive contexts.

# RAG System Functionality
- The RAG system leverages in-context learning to enhance the responses of large language models (LLMs) by incorporating additional context from personal data, such as expense sheets or book content. However, due to the **limited context window of LLMs**, it is crucial to filter and paste only relevant sections of data to avoid exceeding capacity. 
- Effective use of RAG involves selectively providing **context tailored to specific queries**.

# RAG Architecture Overview

- The RAG architecture involves **Dividing knowledge source**, such as a book, into smaller parts, **Generating embeddings** for each part to capture semantic meaning, and storing these embeddings in a specialized **vector store**. 

- When a user asks a question, the query is converted into an embedding and compared against stored vectors to find relevant content. The final response is generated by an LLM based on the context of the relevant pages and the original query.

# Coding RAG Integration
- Starting with the coding section, various methods to implement RAG within LangGraph , focusing on a simple approach that defines **RAG as a tool**. 
- The process includes loading a document, generating embeddings, and storing them in a vector store, crucial for future querying. Finally, a retriever is created based on *similarity searches* for handling user queries about the document.

# RAG Tool Creation
- The RAG tool involves wrapping a retriever function for querying uploaded documents, enabling efficient retrieval of relevant information. 

- By invoking the retriever with a question, it converts the query into an embedding, fetches related vectors from a vector store, and compiles context and metadata into a structured dictionary for further use. 
- This setup lays the groundwork for integrating the tool with an LLM to enhance its functionality.

# Chatbot Interaction Flow
The flow of a RAG-based chatbot involves assessing whether to answer a question directly or retrieve context from a tool called RAG. Upon receiving a query, the chatbot analyzes if additional context is needed before generating a response. The process involves retrieving relevant documents and context, which is then used to formulate the final answer effectively.
